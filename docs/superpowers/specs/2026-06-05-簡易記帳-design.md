## 1. 目的與背景

Ming 需要一個**很簡單**的個人記帳工具，核心訴求是**控制花費、看消費趨勢**，而不是逐筆記帳。

使用方式：每月輸入各帳戶餘額一次，程式自動推算當月消費，並呈現月平均、中位數等趨勢指標，協助判斷花費有沒有變多。

平台：**安卓手機**，以 Chrome「安裝」成 PWA 使用。

## 2. 成功標準

- 每月只需輸入各帳戶餘額（加少量欄位），即可自動算出當月消費，不必逐筆記帳。
- 報表頁能看到月平均消費、月消費中位數、最高/最低、長條圖趨勢。
- 單月消費明顯偏高時有 ⚠️ 視覺提醒。
- 可在安卓 Chrome「安裝」成主畫面 App，離線可開。
- 資料只存本機、不外流；可手動匯出/匯入備份，並在久未備份時提醒。

## 3. 核心計算邏輯

### 消費公式（每月獨立計算）

```
當月消費 =（上月總餘額 ＋ 本月薪資 ＋ 本月額外收入）
        −（定期定額扣款 ＋ 本月額外買基金）
        −（本月總餘額）
```

- 「上月」= 在已記錄月份中，月份字串小於當月的最後一個（非單純日曆上個月，容許跳月）。
- 第一個有記錄的月份沒有「上月」，視為**基準月**，不計算消費。
- 基金（定期定額＋額外）一律視為**投資、不算消費**。
- 額外收入（獎金、退稅、別人還錢等）計入，避免被誤判成「省很多」或出現負消費。

### 異常提醒邏輯

- 計算所有「可計算月份」的消費，求得月平均。
- 若某月消費 > 月平均 × `alertRatio`（預設 **1.2**，設定頁可調），該月在長條圖與明細標 ⚠️ 並用警示色。
- 判斷是相對於使用者整體習慣的動態門檻，非固定金額。

### 統計指標（報表頁）

- 月平均消費、月消費中位數（排序取中）、最高月消費、最低月消費。
- 累計投資 = 所有月份的（定期定額 ＋ 額外基金）總和。
- 目前總資產 = 最新月份的各帳戶餘額加總。

## 4. 資料模型（localStorage）

key：`simple-ledger-v1`

```jsonc
{
  "settings": {
    "salary": 0,          // 每月薪資預設
    "fund": 0,            // 定期定額預設
    "accounts": ["台新", "國泰"],  // 帳戶名稱清單
    "alertRatio": 1.2,    // 異常提醒門檻倍數
    "lastBackup": "2026-06"  // 上次匯出備份的月份；無則 null
  },
  "records": {
    "2026-06": {
      "balances": { "台新": 50000, "國泰": 30000 },
      "salary": 60000,    // 當月薪資（預設帶 settings.salary，可改）
      "fund": 10000,      // 當月定期定額（預設帶 settings.fund，可改）
      "extra": 0,         // 當月額外買基金
      "income": 0         // 當月額外收入（獎金/退稅/還錢）
    }
  }
}
```

- 帳戶改名 / 刪除時，需同步更新所有 `records[*].balances` 的鍵。
- 資料永不上傳，GitHub 上只有程式碼。

## 5. 架構與檔案結構

做法 A「就地升級」：以單一 `index.html`（內聯 CSS/JS）為主體，外加 PWA 最小檔案。純前端、零後端、零第三方套件。

```
記帳程式/
├── index.html      # 主程式（內聯 CSS/JS）
├── manifest.json   # PWA 設定：名稱「簡易記帳」、圖示、直立、深色主題
├── sw.js           # Service Worker，快取優先，離線可用
├── icons/
│   ├── icon-192.png
│   └── icon-512.png
└── docs/superpowers/specs/2026-06-05-簡易記帳-design.md  # 本文件
```

- Service Worker 採「快取優先」：首次連網開啟後快取資源，之後離線可用。
- 主檔名用 `index.html`（GitHub Pages 預設首頁，網址較乾淨）。
- 既有 `記帳程式.html` 原型將更名 / 整併為 `index.html`。

## 6. 介面與操作流程

底部三分頁導覽：記錄 / 報表 / 設定。

### 📝 記錄頁
- 欄位：月份 → 各帳戶餘額 → 薪資（帶預設）→ 定期定額（帶預設）→ 額外買基金 → **額外收入（新增，預設 0）**。
- 即時預覽：本月總餘額、預估消費；超過平均 ×`alertRatio` 顯示「⚠️ 本月消費偏高」。
- 快速概況：已記錄月數、可算消費月數、月平均、中位數。

### 📊 報表頁
- 四格統計：平均 / 中位數 / 最高 / 最低。
- 累計投資、目前總資產。
- 長條圖：偏高月份標 ⚠️ 並用警示色。
- 明細表：月份｜總餘額｜投資｜消費（偏高標 ⚠️）｜刪除。

### ⚙️ 設定頁
- 每月薪資、定期定額預設值。
- 帳戶管理（新增 / 改名 / 刪除）。
- **異常提醒門檻**（預設 1.2，可調）。
- 資料備份：匯出 JSON / 匯入 JSON / 清除全部。
- **備份提醒**：距上次匯出超過 2 個月，開啟時跳一次溫和提醒。

### 安裝引導
- 首次以瀏覽器開啟、且尚未安裝時，底部跳「📲 把我加到主畫面」提示（透過 `beforeinstallprompt` 事件，安卓 Chrome 支援）。

## 7. 邊界情況與錯誤處理

- **基準月**：最早月份不算消費，明細顯示「基準月」。
- **負消費**：餘額不減反增（額外收入未登錄等），照實顯示並提示可能有未登錄收入。
- **帳戶為空**：記錄頁提示先到設定新增帳戶，禁止儲存。
- **localStorage 解析失敗**：回退為初始空資料，不讓程式崩潰。
- **匯入格式錯誤**：驗證須含 `settings` 與 `records`，否則跳錯誤訊息、不覆蓋現有資料；覆蓋前二次確認。
- **跳月**：上月以「已記錄月份中小於當月的最後一個」為準，容許中間缺月。

## 8. 部署

- 整包推到 Ming 指定的**某個 GitHub 帳號**（部署時以互動流程確認，非預設帳號），開啟 GitHub Pages 取得 https 網址。
- 安卓 Chrome 開啟網址後「安裝應用程式」。
- 免費 Pages 程式內容公開，但記帳資料僅存本機、不上傳，隱私無虞。

## 9. 不做（YAGNI）

- 不做逐筆消費分類 / 帳本。
- 不做後端、帳號系統、雲端同步。
- 不做多幣別、匯率。
- 不引入框架或 build 工具。
